クエリビルダーを用いると、データのクエリを実行して、カスタムチャートやその他の可視化を作成することができます。クエリビルダーは、次の 3 つの作業モードを備えています：[基本的なクエリを行わないモード](/docs/chart-builder/use-chart-builder/choose-data/use-basic-mode-specify-data)、[詳細（NRQL）モード](/docs/chart-builder/use-chart-builder/choose-data/use-advanced-nrql-mode-specify-data)、詳細な **PromQL スタイル**モード**。**このモードでは、PromQL クエリを[メトリックス](/docs/using-new-relic/data/understand-data/new-relic-data-types#metrics)の評価やチャート作成の情報として使用します。

## PromQL スタイルモードでクエリビルダーを使用する [#use-promql]

[one.newrelic.com](https://one.newrelic.com/)に移動し、右上隅にある**データのクエリ**をクリックして、クエリビルダーの詳細（PromQL スタイル）モードにアクセスできます。左にあるクエリビルダーのタブに移動し、右側の**PromQL スタイル**を選択します。

<Callout variant="tip">
  当社は、PromQLスタイルクエリモードを通じて、Prometheusクエリ言語（PromQL）をサポートしています。PromQL構文を最も近いNRQL近似に自動的に翻訳するよう努めています。翻訳のしくみと、PrometheusとNew
  Relicの違いの詳細については、[サポートしているPromQL機能](/docs/integrations/prometheus-integrations/view-query-data/supported-promql-features)をご覧ください。
</Callout>

![切り替え後のチャートビルダでのPromQLスタイル表示](src/images/PromQL-after.png 'PromQLモード')

<figcaption>
  **[one.newrelic.com](http://one.newrelic.com)**に移動し、CTRLキーとEキーを押してクエリビルダーを開いてから、**PromQLスタイル**をクリックしてPromQLスタイルクエリを作成します。
</figcaption>

次のパラメーターで PromQL スタイルクエリを構築：

- **アカウント**: クエリを行うアカウントを選択します。
- **PromQL スタイルプロンプト**: ここでクエリを作成します。メトリックスの有効なオプションと、アカウントの機能の一覧を表示できます。
- **手順と範囲**: クエリの手順と範囲を選択するか、インスタントをクリックします。

クエリを実行し、結果と共にチャートを取得します。チャートに名前を付けたり、チャートのタイプを変更したり、ダッシュボードに追加できます。

<Callout variant="tip">
  [NRQL](/docs/query-your-data/explore-query-data/chart-builder/use-advanced-promql-mode-specify-data)をクリックして、PromQLスタイルクエリがどのようにNRQLに翻訳されるかを確認します。
</Callout>

## PromQL スタイルとクエリビルダー [#notes-promql]

詳細な PromQL スタイルモードのクエリビルダーは、クエリ実行前に、[PromQL クエリ](https://prometheus.io/docs/prometheus/latest/querying/basics/)を NRQL に翻訳します。結果は、Prometheus のクエリ実行とは異なる場合があります。

翻訳されたクエリのトラブルシューティング情報を含む、New Relic の PromQL スタイルのクエリ言語の詳細やその動作について知りたい場合、[サポートされた PromQL 機能](/docs/integrations/prometheus-integrations/view-query-data/supported-promql-features)をご覧ください。
