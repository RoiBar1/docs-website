## ヘッダーとフッター [#headers-footers]

ヘッダーやフッターを実装するには

1. &lt;div> タグで囲まれたコンテンツで構成されたヘッダーおよびフッターページを作成してください。ページには CSS を取り込んだり、インライン CSS が含まれないようにしてください。

2. ヘッダーとフッターに必要な CSS を取り、別の CSS ファイルで利用できるようにします。例えば、 `yourcompany_` のようなカスタム文字列を様々なスタイル名の前に置き、New Relic の標準スタイル定義を上書きしないようにします。

3. 3 つのファイルを、パートナーがホストする一般にアクセス可能な場所に投稿します。お客様に混合コンテンツの警告が表示されないように、https を使用してください。例えば、以下のようになります。

   ```
   https://yourcompany.com/newrelic_header.htm
   https://yourcompany.com/newrelic_footer.htm
   https://yourcompany.com/newrelic_styles.css
   ```

4. New Relic **Partnership** コンソールの適切なフィールドに、ヘッダー、フッター、CSS ページの場所の URL を入力します。

<Callout variant="tip">
  ヘッダーを表示するには、パートナーシップのいずれかのアカウントにログインします。
</Callout>

## レイアウトとスタイル [#styles]

ヘッダーとフッターのレイアウトとスタイルは、New Relic のコレクターサーバーに読み込まれてキャッシュされ、各ページ出力に挿入されます。パートナーのサイトのレイアウトに加えられた更新は、1 時間ごとに自動的にリフレッシュされます。

パートナーのウェブサイトへのカスタマイズされたリンクをレイアウトに追加することができます。シングルサインオンが有効な場合、パートナーは顧客固有の情報を New Relic ユーザーセッションに注入し、レイアウトで使用することができます。詳細については、 [シングルサインオンとアクセスコントロール](/docs/accounts-partnerships/partnerships/partner-integration-guide/single-sign-access-control) を参照してください。

## サブドメイン

パートナー様は、オプションでカスタムサブドメインを表示することができます。たとえば、 **yourcompany.newrelic.com** のように、New Relic アカウントにアクセスするお客様に表示することができます。これを手配するには、New Relic の Business Enablement チームにご連絡ください。
