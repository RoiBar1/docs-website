当社の ECS インテグレーションは、 [Amazon ECS](https://docs.aws.amazon.com/ecs/index.html) 環境からのパフォーマンスデータをレポートし、表示します。ECS インテグレーションは他のインテグレーションと連携しているため、 [ECS 上で稼働しているサービスを監視することもできます](/docs/integrations/host-integrations/host-integrations-list/monitor-services-running-amazon-ecs) 。

## 機能

[Amazon Elastic Container Service](https://docs.aws.amazon.com/ecs/index.html) (ECS)は、Amazon EC2 クラスター上の Docker コンテナの実行、停止、管理を簡単に行うことができるスケーラブルなコンテナ管理サービスです。当社の ECS インテグレーションは、ECS オブジェクトからメトリクスをレポートすることで、基礎となるコンテナ・インスタンス（EC2 ローンチ・タイプ）とコンテナ・レイヤーを計測します。この統合により、ECS インスタンス、タスク、サービス、コンテナに対する洞察が得られます。

特徴は以下の通りです。

- 事前に作成されたダッシュボードにデータを表示し、ECS 環境を即座に把握することができます。
- [クエリビルダー](/docs/chart-builder/use-chart-builder/get-started/introduction-chart-builder) 自動報告されたデータから、独自のクエリやチャートを作成します。
- ECS データで [警告条件](/docs/ecs-integration-recommended-alert-conditions) を作成します。
- [New Relic Explorer](/docs/new-relic-one/use-new-relic-one/ui-data/new-relic-one-entity-explorer-view-performance-across-apps-services-hosts) を使用してエンティティを調査します。

<Callout variant="important">
  New Relic では、 [ECS
  クラウドインテグレーション](/docs/integrations/amazon-integrations/aws-integrations-list/aws-ecsecr-monitoring-integration)
  も提供しており、こちらはオンホストのインテグレーションとは異なるデータセットをレポートします。ECS
  を完全に監視するには、両方の統合を有効にすることをお勧めします。
</Callout>

## 互換性および要件 [#requirements]

要件：

- Amazon ECS コンテナエージェント 1.21.0 以降。
- Windows には対応していません。
- この統合では、当社の [インフラストラクチャエージェント](/docs/infrastructure/new-relic-infrastructure/getting-started/compatibility-requirements-new-relic-infrastructure) と [Docker instrumentation](/docs/infrastructure/install-configure-manage-infrastructure/docker-installation/docker-instrumentation-infrastructure) を使用しています：これらのシステムの該当する要件と制限が適用されます。
