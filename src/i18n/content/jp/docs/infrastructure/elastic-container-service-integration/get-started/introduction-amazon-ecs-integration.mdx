当社のECSインテグレーションは、 [Amazon ECS](https://docs.aws.amazon.com/ecs/index.html) 環境からのパフォーマンスデータをレポートし、表示します。ECSインテグレーションは他のインテグレーションと連携しているため、 [ECS上で稼働しているサービスを監視することもできます](/docs/integrations/host-integrations/host-integrations-list/monitor-services-running-amazon-ecs) 。

## 機能

[Amazon Elastic Container Service](https://docs.aws.amazon.com/ecs/index.html) (ECS)は、Amazon EC2クラスター上のDockerコンテナの実行、停止、管理を簡単に行うことができるスケーラブルなコンテナ管理サービスです。当社のECSインテグレーションは、ECSオブジェクトからメトリクスをレポートすることで、基礎となるコンテナ・インスタンス（EC2ローンチ・タイプ）とコンテナ・レイヤーを計測します。この統合により、ECSインスタンス、タスク、サービス、コンテナに対する洞察が得られます。

![New Relic ECS インテグレーション ダッシュボード](./images/new-relic-ecs-integration-dashboards_0.png "new-relic-ecs-integration-dashboards.png")

<figcaption>
  **[one.newrelic.com](https://one.newrelic.com) > Explorer> ECSダッシュボード：** ECSインテグレーションは、Amazon ECSコンテナに関するパフォーマンスデータをレポートします。
</figcaption>

特徴は以下の通りです。

* 事前に作成されたダッシュボードにデータを表示し、ECS環境を即座に把握することができます。
* [クエリビルダー](/docs/chart-builder/use-chart-builder/get-started/introduction-chart-builder) 自動報告されたデータから、独自のクエリやチャートを作成します。
* ECSデータで [警告条件](/docs/ecs-integration-recommended-alert-conditions) を作成します。
* [New Relic Explorer](/docs/new-relic-one/use-new-relic-one/ui-data/new-relic-one-entity-explorer-view-performance-across-apps-services-hosts) を使用してエンティティを調査します。

<Callout variant="important">
  New Relic では、 [ECS クラウドインテグレーション](/docs/integrations/amazon-integrations/aws-integrations-list/aws-ecsecr-monitoring-integration) も提供しており、こちらはオンホストのインテグレーションとは異なるデータセットをレポートします。ECS を完全に監視するには、両方の統合を有効にすることをお勧めします。
</Callout>

## 互換性および要件 [#requirements]

要件：

* Amazon ECSコンテナエージェント1.21.0以降。
* Windowsには対応していません。
* この統合では、当社の [インフラストラクチャエージェント](/docs/infrastructure/new-relic-infrastructure/getting-started/compatibility-requirements-new-relic-infrastructure) と [Docker instrumentation](/docs/infrastructure/install-configure-manage-infrastructure/docker-installation/docker-instrumentation-infrastructure) を使用しています：これらのシステムの該当する要件と制限が適用されます。
