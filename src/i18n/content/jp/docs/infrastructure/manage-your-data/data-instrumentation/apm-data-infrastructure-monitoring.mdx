[APM](/docs/apm/new-relic-apm/getting-started/welcome-new-relic-apm)と[Infrastructure](/docs/infrastructure/new-relic-infrastructure/getting-started/welcome-new-relic-infrastructure)データのインテグレーションにより、APM データと Infrastructure データを並べて表示し、問題の根本的な原因を迅速に発見できます。

Infrastructure モニタリングで APM データを検索・使用する主な方法は次のとおりです。

- [インフラストラクチャモニタリング UI ページで APM チャートを表示する](#apm-chart)
- [ホストをアプリケーションデータ別にフィルタリングする](#filter-by-app)
- [Infrastructure と APM を切り替える](#switch)
- [インベントリとイベントページで APM データを調べる](#inventory-events)

Infrastructure データは、APM の[概要](/docs/apm/applications-menu/monitoring/apm-overview-page)ページにあるホストテーブルの APM に表示されます。

## APM およびインフラストラクチャデータのログを表示 [#logs-context]

また、ログとアプリケーションのデータを統合して、トラブルシューティングをより簡単かつ迅速にできます。[コンテキストのログ](/docs/logs/logs-context/configure-logs-context-apm-agents/)では、エラーとトレースに関連するログメッセージをアプリケーションの UI で直接表示できます。また、Kubernetes クラスタなど、[インフラストラクチャデータ](/docs/logs/forward-logs/forward-your-logs-using-infrastructure-agent/)のコンテキストでログを表示することもできます。New Relic One で別の UI ページに切り替える必要はありません。

## APM と Infrastructure データを統合する方法 [#how-it-works]

APM と Infrastructure データを統合するには、次の条件をすべて満たす必要があります。

- APM エージェントと Infrastructure エージェントは同一のホストにインストールされていること
- 両エージェントが同一の[New Relic ライセンスキー](/docs/apis/intro-apis/new-relic-api-keys/#ingest-license-key)を使用していること
- 両エージェントが[同一のホスト名](/docs/infrastructure/new-relic-infrastructure/troubleshooting/apm-data-missing-infrastructure#hostnames)を使用していること

インテグレーションがうまくいかない場合は、[APM-Infrastructure インテグレーションのトラブルシューティング](/docs/infrastructure/new-relic-infrastructure/troubleshooting/apm-data-missing-infrastructure)をご覧ください。

## APM チャートを表示する [#apm-chart]

お使いの APM と Infrastructure データがリンクされていると、Infrastructure モニタリング UI ページ（**Hosts**、**Network**、**Storage**、および**Processes**）で APM データチャートにアクセスできます。

別のチャートに切り替えるには：チャートの名前の横にあるドロップダウンを選択し、新しいチャートを選択します。アプリケーション関連のチャートは一番上に表示されます。

## アプリケーションデータ別にフィルタリングする [#filter-by-app]

お使いの APM データと Infrastructure データがリンクされていると、**アプリケーション**を使って表示されたホストデータをフィルタリングできます。

- [host filter](/docs/infrastructure/new-relic-infrastructure/filter-group/filter-sets-organize-your-infrastructure-hosts)で**Applications**を選択します。

- フィルタを適用するアプリケーションを選択します。

  <Callout variant="tip">
    **Hosts**ページでは、**Applications**カラムのアイテムを選択することでもフィルタリングが可能です。
  </Callout>

## Infrastructure と APM を切り替える [#switch]

お使いの APM アカウントと Infrastructure アカウントがリンクされていると、同一の時間範囲で Infrastructure から APM へ、あるいはその反対へ切り替えることができます。

Infrastructure から APM への切り替えは、次の場所から行うことができます。

- [Host filter](/docs/infrastructure/new-relic-infrastructure/filter-group/filter-sets-organize-your-infrastructure-hosts)の**Applications**メニューから
- **Host**ページで、**Applications**テーブルカラムのアプリケーションを選択する時。

APM から Infrastructure への切り替えは、APM[概要](#infra-apm-overview)ページにあるホストテーブルから行うことができます。

## インベントリとイベント内の APM データ [#inventory-events]

お使いの APM データと Infrastructure データがリンクされていると、Infrastructure[**インベントリ**ページ](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-inventory-page-search-your-entire-infrastructure)と[**イベント**ページ](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-events-page-live-feed-every-agent)で、アプリケーションデータの表示やフィルタリングができます。

## APM でホストデータを表示する [#infra-apm-overview]

お使いの APM データと Infrastructure データがリンクされていると、APM でより多くのホストデータを利用できます。

APM[概要](/docs/apm/applications-menu/monitoring/apm-overview-page)ページにはお使いのアプリケーションのホストとインスタンスに関する次のデータが、テーブル形式で記載されています。

- Apdex
- レスポンスタイム
- スループット
- エラー率
- CPU 使用率
- メモリ

個々のホストのテーブル表示と分割したメトリックの詳細を切り替えるには、**View table**あるいは **Break out each metric by host**を選択します。

APM**概要**ページのホストデータについての詳細は、[ホストの詳細](/docs/apm/applications-menu/monitoring/apm-overview-page#host_details)をご覧ください。

## APM データの欠落をトラブルシューティングする [#troubleshooting]

APM エージェントと Infrastructure エージェントが同一のホストにインストールされており、両者に同一の[New Relic ライセンスキー](/docs/apis/intro-apis/new-relic-api-keys/#ingest-license-key)と同一のホスト名が使われている場合、APM/Infrastructure インテグレーションは自動的に行われます。Infrastructure モニタリングで APM データが表示されない場合は、[トラブルシューティング](/docs/infrastructure/new-relic-infrastructure/troubleshooting/troubleshooting-apm-data-not-infrastructure)をご覧ください。
