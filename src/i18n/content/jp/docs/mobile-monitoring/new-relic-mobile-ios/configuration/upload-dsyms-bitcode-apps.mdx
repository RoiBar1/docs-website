アプリの dSYM ファイルは、Xcode の dSYM archive path フォルダに格納されています。これは、クラッシュレポートのシンボリック化に使用される dSYM ファイルを iOS エージェントが取得するフォルダです。

New Relic では、 [iOS エージェントのインストールプロセス](/docs/mobile-monitoring/mobile-monitoring-installation/getting-started/ios-installation-configuration) および [tvOS エージェントのインストールプロセス](/docs/mobile-monitoring/new-relic-mobile-ios/get-started/tvos-installation-configuration) の一部として、ポストビルドスクリプトを提供しています。このスクリプトは、アプリの dSYM ファイルを自動的にアップロードします。

## 自動スクリプト [#Automatic]

このスクリプトは、リリースビルドのみ、dSYM ファイル **を自動的にアップロードします** 。リリースビルド以外の場合は、 [手動で](#manual-dsym) または [モバイルモニタリング UI](#mobile-ui) を使ってファイルをアップロードする必要があります。ビットコード対応のアプリは、Apple が dSYM ファイルを生成します。 [Bitcode 対応アプリの dSYM ファイルを Apple からダウンロードして](/docs/mobile-monitoring/new-relic-mobile-ios/install-configure/retrieve-upload-dsyms-bitcode-enabled-apps) New Relic にアップロードする必要があります。

[**Crashes** ページに読めないマシンコードが表示される場合](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/mobile-apps-crashes-dashboard) 、dSYM ファイルが正しくアップロードされていない可能性があります。場合によっては、 [手動で dSYM ファイル](#manual-dsym) をアップロードする必要があります。

自動スクリプトは Python 2 を使用しています。2019 年 10 月現在、 [macOS 10.15 (Catalina)](https://developer.apple.com/documentation/macos_release_notes/macos_catalina_10_15_beta_10_release_notes#3318248) 、Python はデフォルトではインストールされません。自動スクリプト（推奨）を使用している場合は、手動で [Python 2](https://www.python.org/downloads/mac-osx/) をインストールする必要があるかもしれません。Homebrew を使用している場合は、 [Python on Homebrew](https://docs.brew.sh/Homebrew-and-Python) を参照してください。

## 不足している dSYM を特定する [#IdentifyingMissingdSYMs]

Bitcode 対応のアプリが App Store のレビューやアドホック配布のために Apple にアップロードされる際、 [モバイル クラッシュ レポート](/docs/mobile-monitoring/new-relic-mobile/getting-started/ios-agent-crash-reporting) が適切にシンボライズされるように、Apple から dSYM を手動でダウンロードして New Relic にアップロードする必要があります。これらの dSYM は、アプリをアップロードしてから数分以内に Xcode の archives organizer からダウンロードできます。この場合、 [アーカイブを見つけ、dSYM をダウンロードする手順に従ってください](#Archive) 。

アプリに dSYM ファイルがない場合、モバイルモニタリングの UI に 3 つのインジケータが表示されます。

- **バナー通知**: [**クラッシュレポート** ページ](/docs/mobile-monitoring/mobile-monitoring-ui/crashes/mobile-apps-crash-report-dashboard) にバナーによる警告が表示されます。警告の内容は以下の通りです。

```
We were unable to locate your dsym.
```

- **アップロードのプロンプト**: **Crash type summary** ページから、dSYM ファイルがない場合は自動的にアップロードのプロンプトが表示されます。
- **マシンコード**: **クラッシュレポート** ページのクラッシュスタックトレースにはマシンコードが表示され、人間が読めるエラーメッセージは表示されません。

## モバイルモニタリング UI による dSYM ファイルのアップロード [#mobile-ui]

New Relic One の UI から直接 dSYM ファイルを簡単にアップロードすることができます。最大ファイルサイズは 600MB です。dSYM ファイルをアップロードするには。

1. [one.newrelic.com](https://one.newrelic.com) にアクセスし、 **Mobile** をクリックします。そして、リストからお使いのアプリを選択します。
2. 閲覧 **衝突分析**.
3. **Crash types** リストから特定のクラッシュを選択します。
4. **Upload dSYM** をクリックします。dSYM を直接ドラッグ＆ドロップするか、コンピュータからファイルを選択することができます。

## dSYM ファイルの手動アップロード [#manual-dsym]

状況によっては、New Relic の dSYM ファイルの自動アップロードが失敗することがあります。dSYM のアップロードを試みて失敗した場合、詳細なメッセージを含むビルドエラーが作成されます。例えば、ネットワーク障害が発生し、dSYM のアップロードが完了しなかった場合、Xcode はエラーを報告します。New Relic の dSYM アップロードの取り扱いについては、 [New Relic's Online Technical Community](https://discuss.newrelic.com/t/relic-solution-ios-dsym-upload-deep-dive/42513) を参照してください。

自動アップロードに失敗した場合は、手動で dSYM ファイルをアップロードすることができます。複数の dSYM ファイルがある場合は、最大ファイルサイズが 600MB の 1 つの zip に収めることができます。以下のコマンドの <var>YOUR_NEW_RELIC_APPLICATION_TOKEN</var> の値は、 `+[NewRelic startWithApplicationToken:]` (in Objective-C) または `NewRelic.start(withApplicationToken:)` (in Swift) で使用されるキーと同じです。

dSYM ファイルを手動でアップロードする場合。

<CollapserGroup>
  <Collapser

    title="Pythonスクリプト経由（エージェントバージョン6.0.0以上）"
  >
    iOSエージェントのバージョン6.0.0以降では、エージェントにはシンボルを自動的に処理してアップロードするPythonスクリプトが含まれています。このスクリプトは、コマンドラインから呼び出すことができます。

    ```
    NewRelicAgent.framework/Resources/generateMap.py "<var>DSYM_ARCHIVE_PATH</var>" "<var>YOUR_NEW_RELIC_APPLICATION_TOKEN</var>"
    ```

  </Collapser>

  <Collapser
    id="command-line-manual-upload"
    title="コマンドラインで"
  >
    コマンドラインから個別のdSYMファイルを手動でアップロードする場合。

    1. 以下のコマンドで、dSYMファイルをZIPします。 <var>{'~'}/ZIPPED_DSYM_PATH</var> を新しい dSYM アーカイブのパスとファイル名に置き換えてください（例： <var>Users/my-name/desktop</var> ）。また、 <var>{'~'}/dSYM_PATH</var> を既存のdSYMファイルのパスに置き換えてください。

       ```
       /usr/bin/zip --recurse-paths --quiet "<var>{'~'}/ZIPPED_DSYM_PATH</var>" "<var>{'~'}/dSYM_PATH</var>"
       ```

    2. dSYMのZIPファイルを以下のコマンドでアップロードします。

       * For **US accounts**:

         ```
         curl -F dsym=@"<var>{'~'}/DSYM_ZIP_PATH</var>" -H "X-APP-LICENSE-KEY: <var>YOUR_NEW_RELIC_APPLICATION_TOKEN</var>" https://mobile-symbol-upload.newrelic.com/symbol
         ```

       * For **EU accounts**:

         ```
         curl -F dsym=@"<var>{'~'}/DSYM_ZIP_PATH</var>" -H "X-APP-LICENSE-KEY: <var>YOUR_NEW_RELIC_APPLICATION_TOKEN</var>" https://mobile-symbol-upload.eu01.nr-data.net/symbol
         ```

  </Collapser>
</CollapserGroup>

## トラブルシューティング [#auto-upload-fail-troubleshooting]

### 自動アップロードの失敗 [#troubleshooting-auto-upload]

dSYM の自動アップロードに失敗した場合、 **Report Navigator** には、完全な形の cURL コマンドがある場合があります。この cURL コマンドを使用して、アップロードを再試行することができます。エラーの内容によっては、 [dSYM 手動アップロードの手順](#manual-dsym) を行う必要があります。

### dSYM の行方 [#troubleshooting-missing-dsyms]

dSYM ファイルが見つからない場合は、Xcode のビルド設定を確認して、ファイルが生成されていることを確認する必要があるかもしれません。ローカルにビルドされたフレームワークは、別のビルド設定を持っており、同様に更新する必要があるかもしれません。

ビルド設定。

```
Debug Information Format : Dwarf with dSYM File
Deployment Postprocessing: Yes
Strip Linked Product: Yes
Strip Debug Symbols During Copy : Yes
```
