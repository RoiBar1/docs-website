New Relic では、[New Relic REST API](/docs/apis/getting-started/introduction-new-relic-apis)を含む、[複数の API](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2)を提供しています。このドキュメントで紹介する[REST API エクスプローラー](https://api.newrelic.com)では、アドミンユーザーおよび[API キー](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key)を持つユーザーは以下を実行できます：

- 利用可能な REST API エンドポイントの参照
- ユーザーインターフェース（API Explorer）内での REST API とのインタラクション
- ドキュメンテーションの随時更新ソースの表示
- API アクション向け cURL コマンドの取得。
- API Explorer の URL をコピー・アンド・ペーストによる、同僚との構成済み API コールの共有

こうして、ご利用のソフトウェアコンポーネントに追加する前に、ソリューションを素早く検索して API コールをテストできます。

## 機能

New Relic の API Explorer には、選択されたアカウント向けにインタラクティブなユーザーインタフェースが含まれています。API Explorer UI は、API コール（**アプリケーション**、**ユーザー**等）ならびに**GET Metric Data**、**PUT (update) Applications**、**DELETE Applications**等の利用可能な関数を一覧化します。

**Parameters**値を入力していくと、これは自動的に**Request**に表示されるため、リクエスト送信前に構文をテストして検証できます。UI は、必須フィールド、フィールド詳細、その種類（整数、フロート、ブール等）、そしてロケーション（パス、クエリ等）を意味します。

API キーの要件に関する情報は、[REST API キー](/docs/apis/rest-api-v2/getting-started/introduction-new-relic-rest-api-v2#api_key)をご覧ください。

## API バージョン 1 との違い [#v1]

この API Explorer が適用される[New Relic REST API バージョン 2](/docs/apm/apis/requirements/new-relic-rest-api-v2-getting-started)は、New Relic に出入りするデータに焦点を当てたものです。バージョン 2 は、New Relic の非推奨となった REST API バージョン 1 を置き換えます。

バージョン 2 と 1 には、いくつかの違いがある点に注意してください：

- データの名前が異なる可能性があります
- v2 の一部の cURL コマンドが、v1 とは異なる可能性があります

<Callout variant="tip">
  New Relicエージェントは異なるAPIを使用しており、API
  Explorerを介してアクセスすることはできません。
</Callout>
